## § 2 — Stage table DDL — `UDM_Stage.DNA.ACCT_cdc`

Per CLAUDE.md "Data Flow (per table)" + CDC engine framework columns. The Stage table is the CDC-comparison destination — the pipeline writes change rows (I/U/D operations) here, then promotes to Bronze.

### 2.1 Canonical DDL (auto-generated by `schema/table_creator.py`)

```sql
-- UDM_Stage.DNA.ACCT_cdc
-- Auto-created on first pipeline run for SourceName='DNA', SourceObjectName='ACCT'
-- Source-column types mapped from Polars dtypes via _polars_dtype_to_sql
-- Framework columns appended per CDC engine contract

CREATE TABLE UDM_Stage.DNA.ACCT_cdc (
    -- Source columns (mirroring DNA.osibank.ACCT structure post-Polars extraction)
    ACCTNBR             BIGINT          NOT NULL,                -- PK; per UdmTablesColumnsList.IsPrimaryKey=1
    CUSTNBR             BIGINT          NOT NULL,
    ACCTTYPE            NVARCHAR(20)    NOT NULL,
    DATEOPENED          DATETIME2(3)    NOT NULL,                -- Oracle DATE → DATETIME2 per B-13
    DATECLOSED          DATETIME2(3)    NULL,
    DATELASTMAINT       DATETIME2(3)    NOT NULL,
    CURRBAL             DECIMAL(18,4)   NULL,
    TAXID               VARCHAR(40)     NULL,                    -- post-tokenization (per PiiColumnList); token VARCHAR(40) per Round 1 § 17 PiiVault.Token + B-1 hash discipline
    EMAIL               VARCHAR(40)     NULL,                    -- tokenized
    PHONE               VARCHAR(40)     NULL,                    -- tokenized
    CHARGEOFFAMT        DECIMAL(18,4)   NULL,
    -- ... ~20 more source columns ...
    
    -- CDC framework columns (per CLAUDE.md "Data Flow" + cdc/engine.py)
    _cdc_operation      CHAR(1)         NOT NULL                 -- 'I'/'U'/'D' per CDC engine
        CONSTRAINT CK_ACCT_cdc_operation
            CHECK (_cdc_operation IN ('I', 'U', 'D')),
    _cdc_valid_from     DATETIME2(3)    NOT NULL,                -- ms-precision per CDC-NOW-MS rule
    _cdc_valid_to       DATETIME2(3)    NULL,                    -- NULL while _cdc_is_current=1
    _cdc_is_current     BIT             NOT NULL                 -- BIT (0/1) per BCP CSV Contract
        CONSTRAINT DF_ACCT_cdc_is_current DEFAULT 1,
    _cdc_batch_id       BIGINT          NOT NULL,                -- joins to General.ops.PipelineBatchSequence
    _row_hash           VARCHAR(64)     NOT NULL,                -- SHA-256 hex per B-1 (full 64 chars, NOT BIGINT)
    _extracted_at       DATETIME2(3)    NOT NULL                 -- ms-precision
        CONSTRAINT DF_ACCT_extracted_at DEFAULT SYSDATETIME(),
    
    -- Primary key (composite — natural PK from source plus CDC-window for history)
    CONSTRAINT PK_ACCT_cdc PRIMARY KEY CLUSTERED
        (ACCTNBR, _cdc_valid_from)
);
GO

-- Indexes per UdmTablesColumnsList.IsIndex=1 rows for Layer='Stage'
-- (Per § 5 below; auto-applied by index_management.py at table creation)
CREATE NONCLUSTERED INDEX IX_ACCT_cdc_window
    ON UDM_Stage.DNA.ACCT_cdc (_cdc_valid_from, _cdc_valid_to)
    INCLUDE (_cdc_is_current);
GO

CREATE NONCLUSTERED INDEX IX_ACCT_cdc_current_PK
    ON UDM_Stage.DNA.ACCT_cdc (ACCTNBR)
    WHERE _cdc_is_current = 1;
GO
```

### 2.2 Framework columns explained

| Column | Type | Purpose | Reference |
|---|---|---|---|
| `_cdc_operation` | `CHAR(1)` `'I'` / `'U'` / `'D'` | Operation flag from CDC engine: Insert / Update / Delete | `cdc/engine.py` |
| `_cdc_valid_from` | `DATETIME2(3)` | When this CDC row became valid (ms-precision) | CDC-NOW-MS rule in CLAUDE.md |
| `_cdc_valid_to` | `DATETIME2(3)` NULL | When this CDC row was superseded (NULL while current) | CDC engine expire-step |
| `_cdc_is_current` | `BIT` | `1` if row is the current-state representation; `0` after supersession | Used by SCD2 promotion to find Stage rows to compare against Bronze |
| `_cdc_batch_id` | `BIGINT` | BatchId from `General.ops.PipelineBatchSequence` for the run that wrote this row | Joins to operational metadata for audit |
| `_row_hash` | `VARCHAR(64)` | Full SHA-256 hex of the row's content (excluding PK + framework columns) | B-1 in CLAUDE.md — VARCHAR(64) NOT BIGINT |
| `_extracted_at` | `DATETIME2(3)` | When extractor wrote this row (ms-precision) | Set by `prepare_dataframe_for_bcp` |

### 2.3 Special handling notes

- **Polars dtype → SQL Server type mapping** per `_polars_dtype_to_sql` (in `schema/table_creator.py`). See § 6 for the canonical mapping table.
- **PII columns** are pre-tokenized BEFORE the CSV write — so Stage stores tokens (VARCHAR(40)), not plaintext. Per B-1, tokens are full-length SHA-256-derived strings.
- **Oracle DATE → DATETIME2(3)** per B-13 — Oracle DATE includes time component; SQL Server `DATE` would truncate.
- **`UInt64` from non-hash sources** — must be `.reinterpret(signed=True) → Int64` per CLAUDE.md "BCP CSV Contract"; SQL Server cannot natively handle unsigned 64-bit.

## § 3 — Bronze table DDL — `UDM_Bronze.DNA.ACCT_scd2_python`

Per CLAUDE.md "Data Flow (per table)" + SCD2 engine framework columns + R7 R-2 SCD2 enhancements.

### 3.1 Canonical DDL (auto-generated by `schema/table_creator.py`)

```sql
-- UDM_Bronze.DNA.ACCT_scd2_python
-- Auto-created on first pipeline run for SourceName='DNA', SourceObjectName='ACCT'
-- Source-column types mapped via _polars_dtype_to_sql; framework columns per SCD2 contract

CREATE TABLE UDM_Bronze.DNA.ACCT_scd2_python (
    -- Source columns (same structure as Stage; mirrors DNA.osibank.ACCT post-extraction)
    ACCTNBR             BIGINT          NOT NULL,
    CUSTNBR             BIGINT          NOT NULL,
    ACCTTYPE            NVARCHAR(20)    NOT NULL,
    DATEOPENED          DATETIME2(3)    NOT NULL,
    DATECLOSED          DATETIME2(3)    NULL,
    DATELASTMAINT       DATETIME2(3)    NOT NULL,
    CURRBAL             DECIMAL(18,4)   NULL,
    TAXID               NVARCHAR(40)    NULL,                    -- tokenized
    EMAIL               NVARCHAR(40)    NULL,                    -- tokenized
    PHONE               NVARCHAR(40)    NULL,                    -- tokenized
    CHARGEOFFAMT        DECIMAL(18,4)   NULL,
    -- ... ~20 more source columns ...
    
    -- SCD2 framework columns (load-time pair — Silver/Gold contract; per SCD2-P1-a)
    _scd2_key           BIGINT IDENTITY(1,1) NOT NULL,           -- IDENTITY surrogate; EXCLUDE from INSERTs per "Do NOT" rule
    UdmHash             VARCHAR(64)     NOT NULL,                -- full SHA-256 hex per B-1
    UdmEffectiveDateTime DATETIME2(3)   NOT NULL,                -- ms-precision arrival into UDM (load-time)
    UdmEndDateTime      DATETIME2(3)    NULL,                    -- NULL while UdmActiveFlag=1; load-time close timestamp otherwise
    UdmActiveFlag       TINYINT         NOT NULL                 -- 0=closed-by-supersession / 1=active / 2=closed-by-delete per SCD2-R4
        CONSTRAINT CK_ACCT_UdmActiveFlag
            CHECK (UdmActiveFlag IN (0, 1, 2))
        CONSTRAINT DF_ACCT_UdmActiveFlag DEFAULT 1,
    UdmScd2Operation    CHAR(1)         NOT NULL                 -- 'I'=Insert / 'U'=Update-close / 'R'=Resurrected / 'D'=Delete-close
        CONSTRAINT CK_ACCT_UdmScd2Operation
            CHECK (UdmScd2Operation IN ('I', 'U', 'R', 'D')),
    
    -- SCD2 framework columns (source-date pair — R7 R-2 business-date semantics; per SCD2-P1-b)
    UdmSourceBeginDate  DATETIME2(3)    NULL,                    -- per-row waterfall from SCD2DateColumns; per SCD2-P1-e the B-4 in-flight marker requires BOTH UdmEndDateTime IS NULL AND UdmSourceEndDate IS NULL (NOT UdmSourceBeginDate)
    UdmSourceEndDate    DATETIME2(3)    NULL,                    -- '2999-12-31' sentinel while active; chained end on close
    
    CONSTRAINT PK_ACCT_scd2_python PRIMARY KEY CLUSTERED (_scd2_key)
);
GO

-- Filtered UNIQUE index — invariant: exactly one active row per PK
-- Per SCD2-P1-c + V-4: prevents duplicate active rows; downstream consumers query by PK + UdmActiveFlag=1
CREATE UNIQUE NONCLUSTERED INDEX UX_ACCT_active_PK
    ON UDM_Bronze.DNA.ACCT_scd2_python (ACCTNBR)
    WHERE UdmActiveFlag = 1;
GO

-- Index for time-range queries (audit + analytical use)
CREATE NONCLUSTERED INDEX IX_ACCT_scd2_window
    ON UDM_Bronze.DNA.ACCT_scd2_python (UdmEffectiveDateTime, UdmEndDateTime)
    INCLUDE (UdmActiveFlag, ACCTNBR);
GO

-- Index for source-date range queries (R7 R-2 business-date contract)
CREATE NONCLUSTERED INDEX IX_ACCT_source_window
    ON UDM_Bronze.DNA.ACCT_scd2_python (UdmSourceBeginDate, UdmSourceEndDate)
    INCLUDE (ACCTNBR, UdmActiveFlag);
GO

-- Optional: per-table indexes from UdmTablesColumnsList.IsIndex=1 rows
CREATE NONCLUSTERED INDEX IX_ACCT_DATEOPENED
    ON UDM_Bronze.DNA.ACCT_scd2_python (DATEOPENED)
    WHERE UdmActiveFlag = 1;
GO

CREATE NONCLUSTERED INDEX IX_ACCT_CUSTNBR
    ON UDM_Bronze.DNA.ACCT_scd2_python (CUSTNBR)
    WHERE UdmActiveFlag = 1;
GO
```

### 3.2 Framework columns explained

| Column | Type | Purpose | Reference |
|---|---|---|---|
| `_scd2_key` | `BIGINT IDENTITY` | Surrogate key; clustered PK; EXCLUDE from INSERT DataFrames + BCP column lists per CLAUDE.md "Do NOT" rule | `scd2/engine.py` |
| `UdmHash` | `VARCHAR(64)` | Row-hash of source-column content; drives change detection | B-1; same algorithm as `_row_hash` in Stage |
| `UdmEffectiveDateTime` | `DATETIME2(3)` | When row became active in UDM (load-time; arrival timestamp) | SCD2-P1-a load-time pair |
| `UdmEndDateTime` | `DATETIME2(3)` NULL | When row was closed (NULL while active) | SCD2-P1-a |
| `UdmActiveFlag` | `TINYINT` (0/1/2) | 1=active / 0=closed-by-supersession / 2=closed-by-delete | SCD2-R4 — three-value semantic; legacy indexes filter on `Flag=2` for delete audits |
| `UdmScd2Operation` | `CHAR(1)` | I/U/R/D — Insert / Update-close / Resurrected / Delete-close | SCD2 engine |
| `UdmSourceBeginDate` | `DATETIME2(3)` NULL | Source business date (per-row waterfall per R7 R-2). **NOTE**: the B-4 in-flight marker requires BOTH `UdmEndDateTime IS NULL` AND `UdmSourceEndDate IS NULL` per SCD2-P1-e — NOT `UdmSourceBeginDate IS NULL` alone | SCD2-P1-b + SCD2-R2-a |
| `UdmSourceEndDate` | `DATETIME2(3)` NULL | Business chain end-date ('2999-12-31' sentinel while active; NULL is B-4 in-flight marker per SCD2-P1-c) | SCD2-P1-b + SCD2-P1-e |

### 3.3 Critical invariants (per CLAUDE.md "Do NOT" rules)

- **`_scd2_key` is IDENTITY** — never include in INSERT DataFrames or BCP column lists
- **`UdmActiveFlag = 2` is `delete-close` only** — NEVER set in insert path; only via `_execute_bronze_updates(label_suffix='delete_close')`
- **`UdmActiveFlag = 0` is `update-close` only** — set by close-of-supersession path
- **`UdmSourceEndDate = NULL` is B-4 in-flight marker** — required `UdmEndDateTime IS NULL AND UdmSourceEndDate IS NULL AND UdmActiveFlag = 0 AND UdmScd2Operation IN ('U','R')` to identify orphan
- **Bronze writes are append-only** — NEVER `TRUNCATE TABLE Bronze`; SCD2 is by-design history-preserving

## § 4 — Variants

### 4.1 `StripSuffix=1` variant (CARDTXN per SS-1)

When `UdmTablesList.StripSuffix = 1`, the table names drop the `_cdc` / `_scd2_python` suffix:

| Layer | Default (StripSuffix=0) | StripSuffix=1 |
|---|---|---|
| Stage | `UDM_Stage.DNA.CARDTXN_cdc` | `UDM_Stage.DNA.CARDTXN` |
| Bronze | `UDM_Bronze.DNA.CARDTXN_scd2_python` | `UDM_Bronze.DNA.CARDTXN` |

DDL is otherwise identical to § 2 + § 3 (same framework columns, same indexes). Per CLAUDE.md SS-1 Do-NOT rule: this is per-table opt-in for tables migrating off the legacy T-SQL pipeline (AuditLog + CARDTXN currently).

### 4.2 Custom name override (`BronzeTableName` / `StageTableName`)

When `UdmTablesList.BronzeTableName = 'CUSTOM_NAME'`:

| Layer | DDL |
|---|---|
| Stage | `UDM_Stage.DNA.{StageTableName or SourceObjectName}_cdc` |
| Bronze | `UDM_Bronze.DNA.{BronzeTableName or SourceObjectName}_scd2_python` |

The OVERRIDE applies independently per layer — you can have a custom Stage name + default Bronze name, or vice versa.

### 4.3 Per-table SCD2 R-2 configuration variants

When `UdmTablesList.SCD2DateColumns IS NULL`, the Bronze table is structurally identical but the `UdmSourceBeginDate` per-row waterfall is disabled — values come from batch-level scalar instead (per SCD2-P1-a fallback). Same DDL; different runtime population behavior.

When `UdmTablesList.ExcludeFromHash` is non-NULL, the listed columns are dropped from `UdmHash` computation (per SCD2-R10.2) — DDL identical; first-run-after-enable triggers a one-time mass-update wave (B-3 class).

## § 5 — Indexes (per `UdmTablesColumnsList.IsIndex=1` rows)

The pipeline reads `UdmTablesColumnsList` at table-creation time + index-management time:

```sql
-- For each row in UdmTablesColumnsList with IsIndex=1:
SELECT ColumnName, IndexName, IndexType, Layer
FROM General.dbo.UdmTablesColumnsList
WHERE SourceName = 'DNA'
  AND TableName = 'ACCT'
  AND IsIndex = 1;
```

→ Pipeline creates the index using the configured `IndexName` + `IndexType` (typically `NONCLUSTERED`; `CLUSTERED COLUMNSTORE` for 100M+ row tables per W-10).

**Framework indexes are NOT in UdmTablesColumnsList** — they're created at table creation time, automatic:
- `PK_<table>_cdc` (Stage) — composite PK on (source PK, `_cdc_valid_from`)
- `IX_<table>_cdc_window` (Stage) — `(_cdc_valid_from, _cdc_valid_to)` for time-range queries
- `IX_<table>_cdc_current_PK` (Stage) — filtered `(source PK) WHERE _cdc_is_current = 1`
- `PK_<table>_scd2_python` (Bronze) — `_scd2_key` IDENTITY clustered
- `UX_<table>_active_PK` (Bronze) — filtered UNIQUE on `(source PK) WHERE UdmActiveFlag = 1` — load-bearing per SCD2-P1-c; downstream consumer dedup invariant
- `IX_<table>_scd2_window` (Bronze) — `(UdmEffectiveDateTime, UdmEndDateTime)`
- `IX_<table>_source_window` (Bronze) — `(UdmSourceBeginDate, UdmSourceEndDate)`

**Per-table data-column indexes** (e.g., `IX_ACCT_DATEOPENED`, `IX_ACCT_CUSTNBR`) come from `UdmTablesColumnsList` per § 5 above — operator-configured for downstream query patterns.

## § 6 — Polars dtype → SQL Server type mapping

Per `schema/table_creator.py` (specifically `_polars_dtype_to_sql`). Canonical conversions:

| Polars dtype | SQL Server type | Notes |
|---|---|---|
| `Int8` | `TINYINT` | -128 to 127 |
| `Int16` | `SMALLINT` | |
| `Int32` | `INT` | |
| `Int64` | `BIGINT` | |
| `UInt8` | `TINYINT` (unsigned domain via app) | |
| `UInt16` | `INT` | SQL Server has no unsigned 16-bit; widen to INT |
| `UInt32` | `BIGINT` | Same reason |
| `UInt64` | `BIGINT` after `.reinterpret(signed=True)` | Per CLAUDE.md BCP CSV Contract |
| `Float32` | `REAL` | |
| `Float64` | `FLOAT` | |
| `Decimal(p, s)` | `DECIMAL(p, s)` | |
| `Utf8` | `NVARCHAR(N)` | N inferred from `.str.lengths().max()` or default to NVARCHAR(MAX) |
| `Date` | `DATETIME2(3)` | Per B-13; Oracle DATE includes time, so promote |
| `Datetime(ms)` | `DATETIME2(3)` | ms-precision per BCP CSV Contract |
| `Datetime(us)` | `DATETIME2(6)` | rare; primarily Polars-internal |
| `Boolean` | `BIT` | Int8 (0/1) per BCP CSV Contract; NEVER True/False string |
| `Categorical` | `NVARCHAR(N)` | Cast to Utf8 before hashing per E-20 |
| `Binary` | `VARBINARY(N)` | Rare; for byte-string columns |

Edge cases per B-13 (Oracle→SQL Server pitfalls):
- Oracle NUMBER without precision → Polars Float64 → SQL Server FLOAT (loses precision if Oracle had > 15 significant digits; cast to VARCHAR2 in source SQL for critical precision columns per E-21)
- Oracle RAW(16) GUID → big-endian byte order vs SQL Server mixed-endian (requires reordering per B-13.4)
- Oracle BLOB/CLOB 4 GB limit vs SQL Server 2 GB (silent truncation possible per B-13.5)
- Oracle VARCHAR2 BYTE semantics vs SQL Server VARCHAR character semantics (multi-byte differences per B-13.6)
